# Amplify An Existing Project

This document explains how to install the Agentic Meta Project into an existing repository. After installation, the client repo is self-contained ā€” you do not need the meta-project checkout to use it.

## Prerequisites

- Python 3.8 or later.
- An existing repository (it does not need to be a git repo, but it helps).
- Access to the Agentic Meta Project source, either checked out locally or as a zip.

## Installation

### Using the installer directly

```bash
python3 scripts/amplify.py /path/to/your-repo
```

The installer inspects the repo and infers the project type (code, design, or planning). Override with flags if the inference is wrong:

```bash
python3 scripts/amplify.py /path/to/your-repo --primary-type code --enabled-facet design
```

### Key options

| Flag | Effect |
|---|---|
| `--primary-type` | Override inferred type: `code`, `design`, or `planning` |
| `--enabled-facet` | Add facets (pass multiple times): `code`, `design`, `planning` |
| `--profile` | Installation profile: `minimal`, `standard` (default), `full` |
| `--tracking` | `tracked` commits files with the repo; `local` adds them to `.gitignore`. If omitted and the target is a git repo, the installer asks interactively. |
| `--force` | Overwrite existing template-managed files |
| `--no-pm-init` | Skip generating starter Project Memory units |
| `--no-claude-adapter` | Skip generating `.claude/skills/` wrappers |

### Example prompts for Claude Code

After checking out the meta project, tell Claude Code:

> Run `python3 /path/to/agentic-meta-project/scripts/amplify.py .` to amplify this repository, then review the installation report.

Or for more control:

> Run `python3 /path/to/agentic-meta-project/scripts/amplify.py . --primary-type code --enabled-facet design` and then read the installation report.

### Example prompts for Codex

> Read the amplify installer at /path/to/agentic-meta-project/scripts/amplify.py, then run it against this repository. Review the installation report at .agents/bootstrap/amplify-installation-report.md afterward.

## What Gets Created

- `AGENTS.md` ā€” thin agent entrypoint wrapper
- `CLAUDE.md` ā€” Claude-specific adapter wrapper (unless `--no-claude-adapter`)
- `.agents/` ā€” the full portable operating system:
  - `project.json`, `adapters/`, `bootstrap/`, `policy/`, `skills/`, `pm/`, `workflows/`, `state/`, `improvement/`, `docs/`
- `.claude/skills/` ā€” Claude-facing skill wrappers (unless `--no-claude-adapter`)
- `.untracked/` ā€” gitignored scratch and session capture space

## Post-Installation Review Checklist

After installation, review these files:

1. **`.agents/project.json`** ā€” confirm the project name, type, and facets match your project.
2. **`.agents/bootstrap/amplify-installation-report.md`** ā€” see what was created, skipped, and inferred.
3. **`.agents/pm/pm-index.md`** ā€” review the starter PM units. They are scaffolded from directory names and may need tightening.
4. **`.agents/state/current-focus.md`** ā€” confirm the initial focus areas make sense.
5. **`.agents/docs/user/getting-started.md`** ā€” your starting point for using the system.

## Safe Re-Runs

Re-run the installer without `--force` to add missing files without overwriting existing content:

```bash
python3 scripts/amplify.py /path/to/your-repo
```

Use `--force` only when you explicitly want template-managed files refreshed to their latest versions. User-modified content in PM units, workflow packets, and state files will be overwritten by `--force`.

## Upgrading Template Docs

When the meta project releases updated template docs, re-run with `--force` to refresh them:

```bash
python3 scripts/amplify.py /path/to/your-repo --force
```

This overwrites template-managed files but does not touch user-created workflow packets, PM units, or state files (those are generated by the installer on first run and then owned by the project). To see what would change before forcing, diff the template directory against your `.agents/` tree.
